# NixOS 26.05 Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Upgrade the flake from NixOS 25.11 → 26.05 (nixpkgs + home-manager), refresh all flake inputs, and audit version-dependent comments without bumping any `stateVersion`.

**Architecture:** Edit `flake.nix` to pin `nixpkgs` and `home-manager` to the 26.05 release branches. Run `nix flake update` to refresh every input lock. Validate with `nix flake check` and `nix eval`. Re-evaluate the two known version-conditional shims (`catppuccin` displayManager option, `opencode.tui` sink) against 26.05 and either delete the shim if upstream now provides the option, or update the comment to mention 26.05. `system.stateVersion` stays at `25.11` per NixOS guidance (it pins the schema the host was originally installed against); `home.stateVersion` bumps to `26.05` per user direction.

**Tech Stack:** Nix flakes, flake-parts, home-manager, sops-nix, catppuccin/nix, DankMaterialShell.

---

## File Structure

Files this plan touches:

- `flake.nix` — bump `nixpkgs` and `home-manager` URLs to the 26.05 release branches
- `flake.lock` — regenerated by `nix flake update`

- `modules/users/rehan/default.nix:149` — bump `home.stateVersion` from `"25.11"` to `"26.05"`

Files this plan deliberately does NOT touch:

- `modules/hosts/one-piece/default.nix:93` — `system.stateVersion = "25.11"` stays (pins originally installed schema)
- `modules/hosts/_template/default.nix:66` — `system.stateVersion = "25.11"` stays (same reason)

Validation host: `one-piece`. All `nix eval`/`nix build` commands target `nixosConfigurations.one-piece`.

- [ ] **Step 1: Check whether the shim is still required**

Temporarily comment out lines 16-20 of `modules/theming/catppuccin.nix` (the `options.services.displayManager.generic.environment = ...` block) and re-run:


Two outcomes:

**(a) Eval succeeds** → shim is no longer needed. Delete lines 14-20 entirely (the `# Compatibility shim:` comment, the `# This option was removed in NixOS 25.11 but catppuccin/nix still references it` line, and the `options.services.displayManager.generic.environment` block). Re-run the eval to confirm clean.

**(b) Eval fails with `The option ... does not exist`** → shim is still needed. Restore the block, then update the comment on line 15 from:

```nix
      # This option was removed in NixOS 25.11 but catppuccin/nix still references it
```

to:


- [ ] **Step 2: Re-run flake check**

Run: `nix flake check --no-build`
Expected: clean.

- [ ] **Step 3: Stage and commit**

If outcome (a):

## Rollback (if something goes wrong)

A single `git revert HEAD~N..HEAD` (where N is the number of commits this plan produced) returns the flake to its 25.11 state. Because `flake.lock` was committed in Task 2 alongside `flake.nix`, the revert pins every input back to the prior locked revisions — no separate `nix flake update` is needed to undo the upgrade.
